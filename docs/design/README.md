# 设计文档说明

`docs/design` 只保留两类文档：

- 当前实现仍然适用、用于解释框架设计取舍的文档
- 非接入型的通用设计说明，例如加密、JSON 抽象等

已经被 `docs/getting-started` / `docs/guides` 替代，或者仍然大量使用旧 API、旧配置键、旧指标口径的历史 Flow 方案，不再放在活跃设计区。此类文档统一迁入 `docs/design/archive/`，仅作为历史决策记录，不作为当前接入与实现依据。
